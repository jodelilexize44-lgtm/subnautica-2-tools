Subnautica 2 Save Backup Checklist

Early Access saves can be affected by patches, crashes, cloud sync, and co-op testing. This page gives a safe backup workflow without publishing unverified save file paths.

Before A Long Session

CheckWhy
Confirm platform and storeSteam, PC Game Pass, Xbox, and cloud saves can behave differently.
Check updates firstPatch notes can change known issues, co-op behavior, or performance advice.
Run a short test worldCo-op groups should test host ownership, reconnects, and guest progress before a long save.
Know your cloud sync statusCloud conflicts can overwrite the version you expected to keep.

Backup Workflow

  1. Close the game fully before copying or moving any save-related files.
  2. Use the store client or platform settings to identify cloud sync status.
  3. If you find a save folder, copy it to a dated backup folder instead of moving it.
  4. Keep one backup before patches and one after a stable session.
  5. Do not delete or replace saves based on random path advice without confirming your platform.

What To Record

  • Store and platform: Steam, PC Game Pass, Xbox, or cloud.
  • Host name and guest names for co-op worlds.
  • Patch/build date before and after the session.
  • Whether reconnects preserve inventory, blueprint, story, and base state.
