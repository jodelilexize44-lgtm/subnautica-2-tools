How This Page Works

Roadmap details should come from official developer posts, store pages, patch notes, or direct current-build testing. If a topic is not sourced, it stays as a watch item instead of a claim.

Patch Review Checklist

  1. Check official update notes first.
  2. Record the patch date, build number if visible, and source link.
  3. Update updates before changing individual guides.
  4. Retest affected workflows: launch, settings, co-op, saves, resource routes, and base planning.
  5. Mark old rows as changed when a patch contradicts them.
